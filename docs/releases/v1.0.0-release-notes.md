# rJournaler_Web v1.0.0 Release Notes

Release date: 2026-03-08
Tag: v1.0.0

## Summary

rJournaler_Web 1.0.0 is the first production release of the web journaling platform, including secure authentication, entry workflow processing, Optimus/Autobot orchestration, weather and LLM metadata stages, import/review workflows, and admin operations tooling.

## Highlights

- Secure auth and account operations:
  - username/password + MFA (TOTP)
  - trusted device support
  - CSRF protections and audit logging
- Entry workflow and queue pipeline:
  - AUTOSAVE/WRITTEN/FINISHED/IN_PROCESS/COMPLETE/FINAL/ERROR stage lifecycle
  - stage-driven processing through Optimus + Autobots
  - metadata groups 0/1/2/3 and metrics finalize stage
- Admin operations and troubleshooting:
  - queue status dashboard and worker controls
  - targeted reprocess with filter presets, dry-run preview, max rows, and batching
  - live Optimus/Autobot log viewer
- Monthly import workflow:
  - upload ZIP, parse monthly files, review, accept/deny, and queue processing
- Weather and analysis integration:
  - weather location presets
  - NOAA/Open-Meteo weather capture into Meta Group 3
  - timeline/simple/deep analysis pages and model-entry drilldown

## Production Hardening Included

- Cross-platform worker control behavior for Windows development and Linux/Docker production.
- OS-aware helper commands on status dashboard.
- OS-aware Python resolver in weather refresh script.
- Action-scoped admin reprocess error handling.
- Improved structured worker logs for claim/requeue/failure paths.
- Runtime weather cache files excluded from version control.

## Breaking/Behavioral Notes

- Correlation Engine has been removed (migration `017_remove_correlation_engine.sql`).
- Legacy correlation cleanup utility moved to:
  - `scripts/legacy/cleanup-correlation-jobs.php`

## Recommended Post-Deploy Verification

- Login and MFA flow (including trusted device behavior).
- Entry create/autosave/finish pipeline end-to-end.
- Admin queue controls and reprocess dry-run/queue actions.
- Worker lifecycle actions (kill/restart/reconcile) on deployment target OS.
- Weather refresh and dashboard cards.
- Log viewer stream updates.
